# Trip Planner - AI Планер за Патувања

Десктоп апликација за Windows изградена во C# која со помош на вештачка интелигенција генерира персонализирани, детални планови за патување - ден по ден, активност по активност.

---

## Опис на проектот

Во време кога планирањето на одмор честопати предизвикува фрустрации поради огромниот број на контрадикторни информации на интернет, часовите поминати во рачно истражување локации и логистичките компликации при поврзување на географски оддалечени точки, оваа апликација влегува во улога на интелигентен личен туристички агент. Наместо корисникот сам да ги пребарува и споредува рестораните, знаменитостите и транспортните рути, неговата единствена задача е да одговори на краток и визуелно привлечен прашалник од осум чекори.

Апликацијата потоа презема сè на себе, создавајќи персонализиран, детален и логички оптимизиран патнички водич за само неколку секунди.

Она што го издвојува овој софтвер од обичните системи за автоматизација е неговата напредна двофазна архитектура која ги комбинира реалните гео-локациски податоци во реално време со моќта на современата вештачка интелигенција.

Веднаш штом корисникот ќе ги внесе своите параметри – дестинација, датуми, буџет, стил на патување, интереси, темпо и начин на транспорт – апликацијата комуницира со Google Maps преку SerpAPI и извлекува реални туристички локации.

За да се избегне непотребно губење време во транспорт, GPS координатите на пронајдените места се обработуваат со Greedy Nearest Neighbor алгоритам кој ги групира најблиските локации.

Филтрираните податоци потоа се испраќаат до Groq LLM (Llama 3.3 70B Versatile), кој генерира комплетен дневен план со активности, транспортни препораки и проценети трошоци.

Благодарение на асинхроното програмирање, сите повици кон надворешните API сервиси се извршуваат во позадина, додека корисничкиот интерфејс останува целосно одговорен.

Генерираниот резултат се прикажува во богато форматиран документ, кој може да се зачува како `.txt` датотека.

Апликацијата исто така поддржува Light и Dark Mode, при што автоматски се ажурираат сите контроли, копчиња и икони.

# TripGeneratorService

Класата **TripGeneratorService** претставува централна компонента на апликацијата. Нејзината задача е да ги обработи сите информации внесени од корисникот, да ги преземе реалните туристички локации, да ги оптимизира според нивната географска близина и да испрати структуриран prompt до Groq AI моделот.

Целиот процес е организиран во неколку помали методи, при што секој метод има јасно дефинирана одговорност.

## GenerateTripAsync()

Методот **GenerateTripAsync()** претставува главната влезна точка на класата и го координира целиот процес на генерирање на патувањето.

При неговото извршување се реализираат следните чекори:

- читање на API клучевите од конфигурацијата;
- генерирање вовед за избраниот град;
- иницијализација на SerpApiService;
- пребарување реални туристички локации;
- отстранување на дупликати;
- избор на најдобро оценетите 30 локации;
- оптимизација на маршрутата;
- креирање AI prompt;
- повик до Groq API;
- враќање на конечниот резултат.

```csharp
public async Task<string> GenerateTripAsync(TripRequest request)
{
    string cityIntro =
        await GenerateCityIntroAsync(...);

    ...

    allActivities = SortByProximity(allActivities);

    string prompt =
        BuildPrompt(request, placesText);

    string tripPlan =
        await CallGroqAsync(groqKey, prompt);

    return cityIntro + "|||" + tripPlan;
}
```

## SortByProximity()

По преземањето на реалните места преку SerpAPI, тие се обработуваат со методот **SortByProximity()**.

Методот користи **Greedy Nearest Neighbor** алгоритам кој секогаш ја избира следната најблиска локација според GPS координатите.

Со ова значително се намалува непотребното движење и транспортот низ градот.

```csharp
private List<Activity> SortByProximity(
    List<Activity> activities)
{
    var next = remaining
        .OrderBy(a =>
            Math.Pow(a.Latitude - current.Latitude, 2) +
            Math.Pow(a.Longitude - current.Longitude, 2))
        .First();
}
```

## CallGroqAsync()

Методот **CallGroqAsync()** ја реализира комуникацијата со Groq API.

Со помош на `HttpClient` се испраќа HTTP POST барање кое содржи:

- моделот `llama-3.3-70b-versatile`;
- AI prompt;
- максимален број токени.

По приемот на одговорот, JSON содржината се десеријализира и се враќа генерираниот текст.

```csharp
private async Task<string> CallGroqAsync(
    string apiKey,
    string prompt)
```

- пресмета дневни трошоци;
